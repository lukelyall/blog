CSS Styling Guide

Introduction

There are a lot of standards on how CSS should by styled and structured, all with benefits and drawbacks. I found it difficult to follow one folder structure completely, so I made my own CSS structure for modularity and easy readability for others. My CSS file structure splits the main CSS into six folders: setup, theme, interactables, elements, components, and review with one master CSS file.

Folder Structure

This is my current folder structure:
/css/
|- 1.setup
|- 2.theme
|- 3.interactables
|- 4.elements
|- 5.components
|- 6.review
|_ master.css

Setup

The first folder, 1.setup, is for the basic general settings of the project and the layers file. If you are unfamiliar with the @layer feature of CSS, I would recommend reading this @layer documentation.
Setup's file structure:
|- 1.setup
| |- base.css
| |_ layers.css

Theme

The 2.theme folder is for setting up variables, any styles that change the visuals of the site such as background color or text color, and media queries.
Theme's file structure:
|- 2.theme
| |- config.css
| |- display.css
| |_ media-queries.css

Interactables

The third folder, 3.interactables, is for anything a user interacts with such as buttons and sliders.
Interactable's file structure
|- 3.interactables
| |_ button.css

Elements

The 4.elements folder is for how HTML elements without classes are going to look. This is different from the 1.setup folder because it focuses on HTML elements like the spacing of the divs and not the generic CSS setup.
Element's file structure:
|- 4.elements
| |- div.css
| |- footer.css
| |- header.css
| |_ section.css

Components

The fifth folder, 5.components, is for elements that need special classes for appearance. This can include CSS for items such as quote blocks, images, or text blocks.
Components' file structure:
|- 5.components
| |- image-card.css
| |- text-block.css
| |_ quote-block.css

Review

The sixth folder, 6.review, should be the least used folder and is for CSS that works, but how it was made is sketchy so it needs another pair of eyes or to be revisited later.
Review's file structure:
|- 6.review
| |_ fix-me.css

Master File

Finally, the master.css file is where all the layers are combined. This file imports all the layers and their respective files and is the one file added to the script tag in the HTML.
This file will look something like this: /* master css file */

@import url("1.setup/font.css");
@import url("1.setup/base.css");

 @import url("2.theme/config.css");
 That's the whole structure. This file structure has helped me keep my CSS clean and allowed for easier readability for others. Everybody likes to use their own way of structuring CSS, but I hope that this helped to give an idea on what you may want to do with your own CSS files.
